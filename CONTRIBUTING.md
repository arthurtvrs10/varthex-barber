# Contribuindo com o projeto

## Padrão de branches

- `main`: versão estável.
- `develop`: desenvolvimento principal.
- `feature/nome-da-feature`: novas funcionalidades.
- `fix/nome-do-bug`: correções.
- `docs/nome-da-doc`: documentação.

## Padrão de commits

Use commits claros:

```txt
feat: adiciona cadastro de barbeiros
fix: corrige validação de horário disponível
docs: adiciona documentação de monetização
refactor: reorganiza módulo de agendamento
test: adiciona testes de comissão
```

## Antes de abrir Pull Request

- [ ] Código compila.
- [ ] Testes passam.
- [ ] Não há dados sensíveis.
- [ ] Documentação foi atualizada.
- [ ] Fluxo foi testado manualmente.
